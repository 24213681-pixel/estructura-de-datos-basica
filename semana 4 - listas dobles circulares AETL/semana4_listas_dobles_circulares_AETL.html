🔗 Conexión con la Semana 3: Listas Enlazadas Simples

La semana anterior implementaste una lista enlazada simple: cada nodo apunta al siguiente, y solo puedes recorrer en una dirección. Descubriste que eliminar un nodo requiere acceso al nodo anterior, lo que obliga a recorrer desde la cabeza. También viste que eliminarFinal() es O(n) incluso con referencia a cola, porque necesitas actualizar el penúltimo nodo. Esta semana, las listas doblemente enlazadas resuelven exactamente estas limitaciones añadiendo un puntero anterior a cada nodo, permitiendo navegación bidireccional y eliminación eficiente desde cualquier posición.

🎬 ESCENARIO: El Reproductor de Música

Contexto: Trabajas en el equipo de desarrollo de SoundWave, una app de streaming musical. Tu líder técnico, el Ing. Andrés Ríos, te convoca a una reunión urgente.
"Tenemos un problema serio con la playlist. Los usuarios se quejan de que no pueden retroceder a la canción anterior — solo avanzar. Nuestro sistema usa una lista enlazada simple para la cola de reproducción, y cuando alguien presiona '⏮ Anterior', tenemos que recorrer toda la lista desde el inicio para encontrar la canción previa. Con playlists de 500 canciones, el retraso es notable."
Andrés continúa:
"Pero eso no es todo. También necesitamos un modo 'Repetir playlist' donde al terminar la última canción, automáticamente empiece la primera, sin interrupciones. Y los usuarios quieren poder navegar en ambas direcciones fluidamente. ¿Qué estructura de datos propones?"
Tu misión: Rediseñar el sistema de playlist usando una lista doblemente enlazada que permita navegación bidireccional eficiente, y explorar cómo una lista circular resolvería el modo de repetición continua.

(Bandura, 2001; Brod et al., 2023)

Del Nodo Simple al Nodo Doble

En la semana 3, tu nodo tenía dos campos: dato y siguiente. El nodo doble añade exactamente un campo más: anterior. Este campo es una referencia al nodo previo, lo que permite navegar "hacia atrás" sin recorrer desde la cabeza.

┌─────────────────────────────────────────────────────────────────────────────┐ │ NODO SIMPLE vs NODO DOBLE │ ├─────────────────────────────────────────────────────────────────────────────┤ │ │ │ NODO SIMPLE (Semana 3): │ │ │ │ ┌──────────────────────────────┬──────────────────────────────┐ │ │ │ DATO │ SIGUIENTE │ │ │ │ ┌───┐ │ ┌───┐ │ │ │ │ │42 │ │ │ ●─┼──────────► │ │ │ │ └───┘ │ └───┘ │ │ │ └──────────────────────────────┴──────────────────────────────┘ │ │ │ │ NODO DOBLE (Semana 4): │ │ │ │ ┌───────────────┬──────────────────────────┬──────────────────┐ │ │ │ ANTERIOR │ DATO │ SIGUIENTE │ │ │ │ ┌───┐ │ ┌───┐ │ ┌───┐ │ │ │ │ ◄──┼─● │ │ │42 │ │ │ ●─┼──► │ │ │ │ └───┘ │ └───┘ │ └───┘ │ │ │ └───────────────┴──────────────────────────┴──────────────────┘ │ │ │ │ En pseudocódigo: │ │ ┌─────────────────────────────────────────────────────────────┐ │ │ │ clase NodoDoble: │ │ │ │ dato: Tipo │ │ │ │ siguiente: NodoDoble (o null si es el último) │ │ │ │ anterior: NodoDoble (o null si es el primero) │ │ │ └─────────────────────────────────────────────────────────────┘ │ │ │ │ Costo extra: 1 puntero adicional por nodo ≈ 4-8 bytes más de memoria │ │ │ └─────────────────────────────────────────────────────────────────────────────┘
Explicación: El nodo doble es idéntico al simple pero con un campo adicional anterior. Este campo permite "retroceder" al nodo previo sin necesidad de recorrer desde la cabeza. El costo es un puntero extra por nodo (4-8 bytes según la arquitectura), lo cual generalmente es aceptable.

Lista Doblemente Enlazada: Estructura Completa

┌─────────────────────────────────────────────────────────────────────────────┐ │ LISTA DOBLEMENTE ENLAZADA: [10, 20, 30, 40] │ ├─────────────────────────────────────────────────────────────────────────────┤ │ │ │ cabeza cola ` │ │ │ │ │ │ ▼ ▼ │ │ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ │ │ │ 10 │◄───►│ 20 │◄───►│ 30 │◄───►│ 40 │ │ │ └──────┘ └──────┘ └──────┘ └──────┘ │ │ null◄─ ─►null │ │ │ │ Detalle de conexiones entre nodos 20 y 30: │ │ │ │ ┌──────────────┐ ┌──────────────┐ │ │ │ ant │20│ sig │ │ ant │30│ sig │ │ │ └──┼──────┼────┘ └──┼──────┼────┘ │ │ │ └─────────────────┘ │ │ │ │ ┌─────────────────┐ │ │ │ └──────┘ (enlace doble)└──────┘ │ │ │ │ Reglas de la estructura: │ │ • cabeza.anterior = null (no hay nada antes del primero) │ │ • cola.siguiente = null (no hay nada después del último) │ │ • Si A.siguiente = B, entonces B.anterior = A (siempre simétrico) │ │ • Si la lista tiene 1 elemento: cabeza = cola = ese nodo │ │ • Si la lista está vacía: cabeza = cola = null │ │ │ └─────────────────────────────────────────────────────────────────────────────┘
Explicación: La lista doblemente enlazada mantiene dos referencias globales: cabeza (primer nodo) y cola (último nodo). Cada par de nodos adyacentes está conectado en ambas direcciones. La regla de simetría (si A.siguiente = B, entonces B.anterior = A) es el invariante más importante de toda la estructura.

¿Por qué necesitamos listas dobles? — Las limitaciones de la lista simple

🔑 Recordatorio de la semana 3

En la lista simple, para eliminar un nodo necesitas acceso al nodo anterior. Pero solo puedes avanzar hacia adelante. Esto causa dos problemas concretos:

  • eliminarFinal() es O(n) — aun con referencia a cola, necesitas encontrar el penúltimo nodo
  • Eliminar un nodo del que ya tienes referencia sigue siendo O(n) — debes buscar su anterior

La lista doble resuelve ambos problemas porque cada nodo ya sabe quién está antes.

Tabla Comparativa: Complejidades

Operación Arreglo Lista Simple Lista Doble
Acceso por índice O(1) O(n) O(n)
Insertar al inicio O(n) O(1) O(1)
Insertar al final O(1) amortizado O(1) con cola O(1)
Eliminar al inicio O(n) O(1) O(1)
Eliminar al final O(1) O(n) O(1) ← mejora clave
Eliminar nodo conocido O(n) O(n) O(1) ← mejora clave
Búsqueda por valor O(n) O(n) O(n)
Memoria por elemento Solo dato Dato + 1 puntero Dato + 2 punteros
⚠️ Clave: "Eliminar nodo conocido" en O(1)

La mejora más poderosa de la lista doble no es solo eliminar al final en O(1). Es que si ya tienes una referencia directa al nodo que quieres eliminar (por ejemplo, la canción actual en la playlist), puedes eliminarlo en O(1) sin buscar su anterior. En la lista simple, aun teniendo el nodo, necesitas recorrer para encontrar el anterior. Esta diferencia es crucial para estructuras como cachés LRU, editores de texto, y navegadores web.

Pseudocódigo Completo: Operaciones Fundamentales

A continuación se presentan las operaciones clave con pseudocódigo comentado línea por línea. Lee cada comentario para entender por qué existe cada línea, no solo qué hace.

┌─────────────────────────────────────────────────────────────────────────────┐ │ PSEUDOCÓDIGO: INSERTAR AL INICIO │ ├─────────────────────────────────────────────────────────────────────────────┤ │ │ │ método insertarInicio(valor): │ │ nuevo ← crear NodoDoble(valor) // Crear nodo con dato = valor │ │ nuevo.siguiente ← cabeza // El nuevo apunta al ex-primero │ │ nuevo.anterior ← null // Nada antes del primero │ │ │ │ SI cabeza ≠ null: // Si la lista NO estaba vacía │ │ cabeza.anterior ← nuevo // El ex-primero apunta atrás │ │ SINO: // Si la lista ESTABA vacía │ │ cola ← nuevo // El nuevo es también el último │ │ │ │ cabeza ← nuevo // Actualizar cabeza │ │ tamaño ← tamaño + 1 │ │ │ │ Complejidad: O(1) — sin importar el tamaño de la lista │ │ │ └─────────────────────────────────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────────────────────────────┐ │ PSEUDOCÓDIGO: INSERTAR AL FINAL │ ├─────────────────────────────────────────────────────────────────────────────┤ │ │ │ método insertarFinal(valor): │ │ nuevo ← crear NodoDoble(valor) // Crear nodo con dato = valor │ │ nuevo.anterior ← cola // El nuevo apunta atrás al cola │ │ nuevo.siguiente ← null // Nada después del último │ │ │ │ SI cola ≠ null: // Si la lista NO estaba vacía │ │ cola.siguiente ← nuevo // El ex-último apunta adelante │ │ SINO: // Si la lista ESTABA vacía │ │ cabeza ← nuevo // El nuevo es también el primero │ │ │ │ cola ← nuevo // Actualizar cola │ │ tamaño ← tamaño + 1 │ │ │ │ Complejidad: O(1) — simétrico a insertarInicio │ │ │ └─────────────────────────────────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────────────────────────────┐ │ PSEUDOCÓDIGO: ELIMINAR UN NODO CONOCIDO │ ├─────────────────────────────────────────────────────────────────────────────┤ │ │ │ método eliminarNodo(nodo): // Recibe referencia directa │ │ │ │ // PASO 1: Reconectar el nodo anterior (si existe) │ │ SI nodo.anterior ≠ null: │ │ nodo.anterior.siguiente ← nodo.siguiente // Saltar nodo │ │ SINO: │ │ cabeza ← nodo.siguiente // Era el primero: mover cabeza │ │ │ │ // PASO 2: Reconectar el nodo siguiente (si existe) │ │ SI nodo.siguiente ≠ null: │ │ nodo.siguiente.anterior ← nodo.anterior // Saltar nodo │ │ SINO: │ │ cola ← nodo.anterior // Era el último: mover cola │ │ │ │ tamaño ← tamaño - 1 │ │ │ │ Complejidad: O(1) — no recorre nada, solo reasigna punteros │ │ │ │ ⚠️ NOTA: Este método asume que "nodo" pertenece a esta lista. │ │ No valida pertenencia. En producción, esto puede ser un bug. │ │ │ └─────────────────────────────────────────────────────────────────────────────┘
Punto clave: Observa cómo eliminarNodo() maneja 4 casos automáticamente: nodo del medio, primer nodo, último nodo, y único nodo. Las dos condiciones (anterior ≠ null y siguiente ≠ null) cubren todas las combinaciones. Este patrón de "verificar ambos lados" es el corazón de la lista doble.

Errores Comunes esta Semana

🚫 Errores frecuentes con listas dobles
  • Olvidar actualizar el puntero anterior: Si insertas un nodo y solo actualizas siguiente pero no anterior del nodo adyacente, la lista queda asimétrica. Parece funcionar al recorrer hacia adelante, pero falla al retroceder.
  • No manejar cabeza Y cola juntas: Si la lista tiene un solo elemento y lo eliminas, debes poner cabeza = cola = null. Olvidar uno de los dos deja un "fantasma".
  • Asumir que el nodo pertenece a la lista: eliminarNodo(nodo) no verifica que el nodo sea parte de esta lista. Si pasas un nodo de otra lista, corrompes ambas.

🔒 Invariantes que no deben romperse — Fase COMPRENDE

  • Si A.siguiente = B, entonces B.anterior = A (simetría bidireccional)
  • cabeza.anterior = null siempre (nada antes del primero)
  • cola.siguiente = null siempre (nada después del último)
  • Si la lista tiene un solo nodo: cabeza = cola = ese nodo
  • Si la lista está vacía: cabeza = cola = null

Elección de Lenguaje

💻 Lenguaje sugerido para implementación

Puedes implementar en el lenguaje que prefieras. Cada opción enseña algo diferente:

  • Python: Sintaxis simple, garbage collector maneja la memoria. Ideal si quieres enfocarte en la lógica de punteros sin distracciones.
  • Java / C#: Clases y genéricos. Aprendes a definir tipos genéricos (NodoDoble<T>), manejo de null references.
  • C / C++: Gestión manual de memoria (malloc/free o new/delete). Aprendes el costo real de los memory leaks.
  • JavaScript: Si ya estás aprendiendo JS en otra materia, puedes aprovechar la sinergia. Los objetos como nodos son naturales.

Lo importante no es el lenguaje, sino entender las decisiones que cada operación obliga a tomar (qué punteros actualizar, en qué orden, y qué pasa si olvidas uno).

Análisis de Trade-offs: ¿Cuándo vale la pena el puntero extra?

La lista doble usa más memoria que la simple (un puntero extra por nodo). No siempre vale la pena. El criterio es: ¿tu aplicación necesita frecuentemente navegar hacia atrás o eliminar nodos de los que ya tienes referencia?

✅ Usa lista doble cuando...
  • Navegación bidireccional es frecuente (playlist, historial de navegador, undo/redo)
  • Necesitas eliminar nodos de los que ya tienes referencia directa en O(1) (caché LRU)
  • Eliminar al final es operación frecuente
⚠️ Quédate con lista simple cuando...
  • Solo necesitas recorrer en una dirección (procesamiento de streams, logs)
  • Memoria es muy limitada y el puntero extra es costoso
  • Solo insertas/eliminas al inicio (pila implementada con lista)

Reflexión Guiada

  2. Sobre memoria: Si cada canción ocupa 200 bytes de metadatos y un puntero ocupa 8 bytes, ¿qué porcentaje de memoria extra usa la lista doble vs. la simple para una playlist de 1,000 canciones? ¿Es significativo?

Verificación Sistemática

✅ Checklist de Autoevaluación Rápida — Verifica antes de entregar

Estos son los errores más frecuentes. Si alguno falla, probablemente tienes un puntero mal actualizado. Márcalos uno por uno:

  • ☐ ¿Imprimir hacia atrás funciona correctamente? — Si entra en bucle infinito → puntero anterior mal actualizado
  • ☐ ¿Borrar el primer elemento actualiza correctamente cabeza? — Si cabeza sigue apuntando al nodo eliminado, la lista queda corrupta
  • ☐ ¿Borrar el último elemento actualiza correctamente cola? — Mismo problema pero al final
  • ☐ ¿Insertar en lista vacía actualiza tanto cabeza como cola? — Olvidar uno de los dos deja un "fantasma"
  • ☐ ¿Después de cualquier operación, la simetría bidireccional se mantiene? — Prueba ácida: si A.siguiente = B, entonces B.anterior = A
  • ☐ ¿El tamaño coincide con el número real de nodos? — Recorre la lista y cuenta

💡 Tip: Si encuentras un error con este checklist, dibuja el estado de la lista antes y después de la operación problemática. El diagrama casi siempre revela qué puntero olvidaste actualizar.

Extensión Opcional: Lista Circular

Una lista circular es una variante donde el último nodo apunta de vuelta al primero (y en la versión doble, el primero apunta atrás al último). No tiene null al final — siempre puedes seguir avanzando.

┌─────────────────────────────────────────────────────────────────────────────┐ │ LISTA DOBLEMENTE ENLAZADA CIRCULAR │ ├─────────────────────────────────────────────────────────────────────────────┤ │ │ │ ┌───────────────────────────────────────┐ │ │ │ │ │ │ ▼ │ │ │ cabeza───►┌───┐ ┌───┐ ┌───┐ ┌───┐ │ │ │ │ A │◄───►│ B │◄───►│ C │◄───►│ D │──────┘ │ │ └───┘ └───┘ └───┘ └───┘ │ │ │ ▲ │ │ └───────────────────────────────────────┘ │ │ │ │ Diferencias con la lista doble normal: │ │ • cola.siguiente = cabeza (en vez de null) │ │ • cabeza.anterior = cola (en vez de null) │ │ • No se necesita referencia "cola" separada: cabeza.anterior ES cola │ │ • CUIDADO: Los recorridos necesitan condición de parada (volver al │ │ nodo inicial), o se crean ciclos infinitos │ │ │ │ Casos de uso: │ │ • Playlist en modo "repetir" (SoundWave) │ │ • Round-robin de procesos en un sistema operativo │ │ • Buffer circular para streaming de datos │ │ • Juegos de mesa (turnos cíclicos entre jugadores) │ │ │ └─────────────────────────────────────────────────────────────────────────────┘
Explicación: En la lista circular, no existen los punteros null. Esto simplifica algunas operaciones (no hay casos especiales de "primero" o "último") pero complica los recorridos: si olvidas la condición de parada, entras en un ciclo infinito. La lista circular doble es especialmente elegante porque cabeza.anterior te da acceso al último nodo en O(1), eliminando la necesidad de una variable cola separada.
💡 Esta extensión es opcional

La lista circular no es obligatoria para la entrega de esta semana. Si tu tiempo es limitado, enfócate en completar y probar bien tu ListaDoble. Si tienes tiempo extra, la circular es una excelente extensión que profundiza tu comprensión y adelanta conceptos útiles para las semanas siguientes.

📖 Glosario

Nodo Doble (Double Node)
Nodo con tres campos: dato, siguiente (referencia al próximo nodo) y anterior (referencia al nodo previo).
Lista Doblemente Enlazada (Doubly Linked List)
Estructura lineal donde cada nodo tiene punteros al siguiente y al anterior, permitiendo recorrido bidireccional.
Lista Circular (Circular List)
Variante donde el último nodo apunta al primero (y en la versión doble, el primero apunta al último). No tiene null.
Simetría Bidireccional
Invariante donde si A.siguiente = B, entonces B.anterior = A. Romper esta simetría corrompe la lista.
Cabeza / Cola (Head / Tail)
Referencias al primer y último nodo de la lista respectivamente. En una lista circular, cola = cabeza.anterior.
Sentinel Node (Nodo Centinela)
Nodo ficticio (sin dato real) usado al inicio o final de la lista para eliminar casos especiales en el código.
Navegación Bidireccional
Capacidad de recorrer una estructura tanto hacia adelante como hacia atrás desde cualquier nodo.
Caché LRU (Least Recently Used)
Estrategia de caché que elimina el elemento menos usado recientemente. Se implementa eficientemente con lista doble + tabla hash.
Trade-off (Compromiso)
Decisión de diseño donde ganar en un aspecto (ej. velocidad de eliminación) implica perder en otro (ej. memoria por nodo).
Caso Frontera (Edge Case)
Situación límite que puede causar bugs: lista vacía, un solo elemento, operar en el primero o último nodo.
Verificación de Integridad
Función que recorre la estructura verificando que todos los invariantes se cumplan. Herramienta esencial de debugging.
Agencia Compartida (AETL)
Balance entre autonomía del estudiante y asistencia de la IA. El estudiante es el Arquitecto; la IA es la herramienta.
